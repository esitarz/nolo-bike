Promotion Handling Strategies: 
Eric, Robert, and Miranda evaluated two approaches for handling promotions—Stripe native and Order Cloud sourced—and decided to use Stripe-managed promotions for the MVP, with considerations for future enhancements.
	Promotion Implementation Options: Eric described two possible methods: using Stripe native promotions, which are managed and applied within Stripe, and Order Cloud sourced promotions, which would require applying the promotion before sending the order to Stripe and potentially overriding line item prices. The Stripe native approach was demonstrated in a proof of concept.
	Order Cloud Promotion Limitations: Miranda pointed out that overriding unit prices in Order Cloud is not an accurate way to apply promotions, as true promotion application involves applying a promo code to the order and adjusting the price accordingly. The team agreed that for MVP, Stripe native promotions are sufficient, and more complex integration can be considered later.
	Reconciling Promotions Between Systems: The group discussed the need to ensure that the order price in Order Cloud matches the final price after Stripe promotions are applied. Miranda noted that there are mechanisms to override promotion discounts on line items in Order Cloud, but these should be used carefully, and for MVP, promo codes could be disallowed if complexity arises.
	Future Considerations: Robert and Miranda agreed that while the current approach is suitable for MVP and smaller customers, the long-term goal is tighter integration with Sitecore's main products and a more robust promotion administration experience.
